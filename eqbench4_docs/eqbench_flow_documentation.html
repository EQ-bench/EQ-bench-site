1What is EQ-Bench?

EQ-Bench 4 evaluates how well AI models handle emotionally complex conversations. Unlike simple Q&A benchmarks, it simulates realistic interactions where a person seeks support for a difficult situation—and the AI must build trust, understand hidden issues, and provide genuine help.

The Core Idea

Imagine someone talking to a therapist, friend, or crisis counselor. They don't reveal everything immediately. They have defenses, trust issues, and underlying problems they might not even recognize. EQ-Bench tests whether an AI can navigate this complexity—building rapport, recognizing what's really going on, and helping the person make progress.

The five phases of an EQ-Bench evaluation

2Dataset Generation

Before any conversations happen, EQ-Bench generates a diverse set of scenarios—each representing a unique person with specific psychological traits, issues, and circumstances. This is done by sampling from a carefully curated registry of psychological building blocks.

The registry provides building blocks; the generator assembles them into complete scenarios

What's in the Registry?

Core Issues

The real underlying problem the person is dealing with, often hidden even from themselves. Categories include:

Mental Health Relationship Trauma Parenting Personal Growth

Presenting Issues

What the person says they need help with. This is the surface-level complaint that masks the core issue.

Example: "I can't sleep" might mask underlying anxiety about job loss.

Persona Traits

Three behavioral dimensions that shape how the person interacts:

  • Trust Style – How they build/withhold trust
  • Defense Style – Psychological defenses (sarcasm, intellectualizing, etc.)
  • Help-Seeking Stance – Genuinely seeking help, testing, or just venting
Modifiers

Demographic and personality factors that add context:

  • Age, gender, sexuality
  • Cognitive style, emotional tendencies
  • Background factors (trauma history, family dynamics)
Example: Why Compatibility Matters

The generator enforces logical constraints. A "parenting burnout" core issue requires a modifier indicating the person has children. A "social anxiety at parties" presenting issue excludes modifiers like "extroverted socialite." This ensures scenarios are psychologically coherent.

The Hidden vs. Visible Distinction

A key design principle: the presenting issue and adjacent attributes are what the persona might naturally share. The core issue is hidden—even from the persona in some cases. Good AI performance means recognizing clues and gently exploring deeper, not just responding to surface complaints.

Weighted Scoring

Not all dimensions are equal. Trust, Safety, and Progress on Core Issue are weighted 3x because they're most indicative of meaningful therapeutic progress. The weights reflect which emotional changes matter most for genuine help.

5Conversation Execution

The heart of EQ-Bench is a 10-turn conversation between the AI being evaluated and a simulated persona. The persona is played by another AI model that follows the scenario's traits and updates its emotional state after each exchange.

Each persona turn includes internal reasoning, state update, and dialogue

What the Persona Outputs Each Turn

[INTERNAL]

Private reasoning about how they feel about the AI's response. Are they building trust? Feeling pushed? Considering opening up?

[STATE]

Updated values for all 20 emotional dimensions as JSON. This is how we track whether the conversation is helping or hurting.

[DIALOGUE]

What the persona actually says to the AI. This is the only part the evaluated model sees.

Perspective Flipping

A clever trick: when calling the persona simulator, the conversation roles are flipped. The persona model sees itself as the "assistant" responding to a "user" (the AI being evaluated). This lets a single model naturally play the persona role while reacting authentically to the evaluated model's responses.

6The Debrief Phase

After the conversation ends, the evaluated model is asked to provide a debrief assessment—a reflection on what they learned about the person. This tests the AI's understanding and attunement.

What the AI Must Provide
Personality Assessment

A written description of what they learned about the person's character, defense mechanisms, and communication style.

Core Issue Assessment

What does the AI think is the real underlying issue? This is compared against the hidden ground truth.

State Assessment

The AI's estimate of where each emotional dimension ended up (0-10 scale for each).

How Debrief is Judged

A separate judge model evaluates the debrief across four dimensions:

Dimension What It Measures Scale
Personality Insight How accurately did the AI understand the person's character and traits? 0-10
Core Issue Insight Did the AI correctly identify the hidden underlying problem? 0-10
Emotional Attunement How close was the AI's state estimate to the actual final state? 0-10
Overall Understanding Holistic assessment of comprehension quality 0-10

Ensemble Judging

Multiple judge models evaluate each conversation, and their scores are averaged. This reduces individual judge bias and produces more reliable assessments. The ensemble can run in "split" mode (round-robin assignment) or "stack" mode (fixed assignment per conversation).

8Final Scoring

The final EQ-Bench score combines three components, each measuring a different aspect of emotional intelligence:

50% User State Change
25% EQ Rubric Score
25% Debrief Score
Final Score = 0.50 User State + 0.25 EQ Rubric + 0.25 Debrief

Component Calculations

User State Score (50% weight)

Measures how much the persona's emotional state improved during the conversation:

  1. Calculate deltas for each dimension (final - initial)
  2. Flip sign for negative dimensions (so decrease = positive delta)
  3. Apply weights (trust: 3x, safety: 3x, progress: 3x, etc.)
  4. Compute weighted average of all deltas
  5. Normalize to 0-10 scale (delta of -1 0, delta of +5 10)
Example

If trust went from 3.0 6.0 (+3.0) and resistant went from 5.0 3.0 (-2.0, but flipped = +2.0), these improvements contribute positively to the score.

EQ Rubric Score (25% weight)

Average of judge scores across the 25 dimensions:

  1. Collect judge scores for all positive dimensions (0-10 each)
  2. Collect judge scores for negative dimensions and invert them (10 - score)
  3. Exclude neutral dimensions from averaging
  4. Average all scored dimensions

Debrief Score (25% weight)

Average of the four debrief judgment dimensions:

Debrief = ( Personality + Core Issue + Attunement + Overall ) 4

Fallback Scoring

If some components are unavailable, the formula adjusts:

Situation Formula
No debrief available 50% User State + 50% EQ Rubric
No EQ rubric available 50% User State + 50% Debrief
No judges at all 100% User State
Why This Weighting?

User State (50%) is weighted highest because it directly measures the impact of the conversation—did the person actually feel better and make progress? EQ Rubric (25%) assesses conversation quality and technique. Debrief (25%) measures understanding and attunement. Together, they capture both effectiveness and skill.
